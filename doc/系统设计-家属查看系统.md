# Jide 家属数据查看系统 - 详细设计文档

## 1. 系统概述

家属数据查看系统允许患者的家人、照护者查看患者的训练数据，提供远程关怀和支持。

---

## 2. 账户体系设计

### 2.1 账户类型
| 类型 | 权限 | 场景 |
|-----|------|-----|
| 患者账户 | 训练、数据本人可见 | 被照护者 |
| 照护者账户 | 查看数据、辅助操作 | 家属 |
| 医生账户 | 查看脱敏数据、导出报告 | 医疗团队 |
| 管理员 | 系统管理 | 运营人员 |

### 2.2 账户关联流程
```
患者A                           照护者B
  │                                │
  │  生成邀请码/二维码             │
  │ ──────────────────────────>   │
  │                                │
  │                          输入邀请码
  │                                │
  │  <─────────────────────────   │
  │  确认关联请求                   │
  │ ──────────────────────────>   │
  │                                │
  │  关联成功 ✓                    │
```

### 3.2 趋势分析
- 周/月/年训练趋势图
- 能力维度变化曲线
- 与上次评估对比
- 异常提醒（如连续未训练）

### 3.3 提醒设置
| 提醒类型 | 设置选项 |
|---------|---------|
| 训练提醒 | 每日时间/每周次数 |
| 未训练提醒 | 超过N天未训练 |
| 异常提醒 | 表现大幅下降 |
| 成就提醒 | 获得新成就 |

### 3.4 鼓励消息
- 预设鼓励语模板
- 自定义消息
- 定时发送/立即发送

## 4. 医生账户功能

### 4.1 数据脱敏
医生看到的患者数据不包含：
- 真实姓名（仅显示ID）
- 账户信息
- 隐私数据

## 5. 隐私控制

### 5.1 数据分享级别
| 级别 | 可见内容 |
|-----|---------|
| 完全公开 | 所有训练数据 |
| 仅看趋势 | 仅趋势图，无明细 |
| 仅看得分 | 仅分数，无详情 |
| 完全隐藏 | 拒绝分享 |

### 5.2 控制面板
```
┌─────────────────────────────────────────┐
│  🔒 数据分享设置                         │
│                                         │
│  照护者B   [完全公开 ▼]  [解除关联]     │
│  照护者C   [仅看趋势]   [解除关联]      │
│  医生D    [仅看得分]   [解除关联]      │
│                                         │
│  + 添加新的分享对象                      │
└─────────────────────────────────────────┘
```

## 6. 技术实现

### 6.1 数据结构
```typescript
// 账户关联
interface AccountRelation {
  id: string;
  patientId: string;      // 患者ID
  caregiverId: string;   // 照护者ID
  relationType: string;  // 关系：子女/配偶/其他
  permissionLevel: 'full' | 'trend' | 'score';
  status: 'pending' | 'active' | 'removed';
  createdAt: Date;
}

// 提醒设置
interface ReminderSettings {
  userId: string;
  trainingReminder: {
    enabled: boolean;
    time: string;        // HH:mm
    days: number[];     // 周几
  };
  absenceAlert: {
    enabled: boolean;
    thresholdDays: number;
  };
}
```

### 6.2 API设计
```typescript
// 生成邀请码
POST /api/relation/invite
{ patientId: string, relationType: string }

// 关联账户
POST /api/relation/bind
{ inviteCode: string, caregiverId: string }

// 获取照护者数据
GET /api/caregiver/dashboard?patientId=xxx

// 发送鼓励消息
POST /api/message/encourage
{ patientId: string, message: string }
```
